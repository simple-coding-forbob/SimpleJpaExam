# spring 코딩법: 순서
# spring 인터넷 기본주소: http://localhost:8080
# 1) entity(db 테이블과 비슷) -> 2) dto -> 3) repository -> 4) service(기능별 함수) -> 4) controller(인터넷주소+화면)

## 1) entity : 테이블의 데이터, 화면 입력값을 자바 클래스에 임시저장할 목적으로 사용합니다.
##           1) 업무 테이블의 컬럼을 참고해서 클래스의 필드를 작성합니다.
             2) 생성자, setter, getter, tostring, equals, hashCode 를 추가합니다. 어노테이션을 이용합니다.
             3) 최초 1번만 작성합니다.
             4) 임시 컬럼 등 변경 불가입니다.
## 2) dto : entity 는 필드 추가/삭제하기 어렵습니다. 
##          그래서 여기에서 임시 필드 또는 화면에 안보여줄 필드를 삭제해서 사용합니다.
##           1) entity 를 참고해서 사용합니다.
             2) 생성자, setter, getter, tostring, equals, hashCode 를 추가합니다. 어노테이션을 이용합니다.
             3) 최초 1번만 작성합니다.
             4) 임시 컬럼 등 변경 가능합니다.
             5) 코딩시에 화면 -> dto -> entity 로 데이터 복사해야 합니다.(반대도 마찬가지 입니다.)
                자동 복사 라이브러리를 사용하면 편합니다.(mapstruct 라이브러리) 

## 3) repository : CRUD 함수를 작성합니다. 기능별 sql 을 작성합니다.
       1) 인터페이스 : 인터페이스를 작성하고 안에 CRUD 함수를 작성합니다.
          JpaRepository<엔티티,기본키필드자료형> 상속 받습니다.
       2) findAll(전체조회), findById(상세조회), save(저장), updateFromDto(수정), deleteById(삭제) 는 
          실행하면 jpa 가 sql 을 자동으로 작성해서 실행합니다.(개발자 작성 필요 없음)
       3) 개발자가 직접 sql 을 작성하는 기능도 있습니다.
        

## 4) service: mapper 의 CRUD 함수를 사용해서 기능별 함수를 작성합니다.
      1) 기본 6개(전체조회,총데이터개수, 추가, 상세조회, 수정, 삭제)를 작성합니다. 상황에 따라 기능을 추가/생략합니다.

## 5) controller: 각 기능별 인터넷 주소를 정하고 해당 화면(리액트)으로 데이터를 전달합니다.
       1) /dept(인터넷 주소) -> json 데이터 -> 리액트(화면)에서 데이터 표시
       2) 함수위에 어노테이션을(@GETMAPPING 등) 이용해서 작성합니다.

# 3) 페이징 : JPA 가 자동으로 처리합니다.
